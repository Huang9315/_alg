# C(n, k) 查表版與動態規劃版之異同

## 相同點
- 皆使用 **Pascal 遞迴公式**  
  \[
  C(n,k) = C(n-1,k-1) + C(n-1,k)
  \]
- 皆滿足邊界條件  
  \[
  C(n,0) = C(n,n) = 1
  \]
- 都透過 **表格儲存中間結果**，避免重複計算
- 時間複雜度約為 **O(nk)**

---

## 不同點

| 項目 | 查表版（Lookup / 記憶化） | 動態規劃版（DP / Tabulation） |
|------|--------------------------|------------------------------|
| 計算方向 | Top-Down | Bottom-Up |
| 實作方式 | 遞迴 + 查表 | 迴圈填表 |
| 表格填寫時機 | 需要時才填（Lazy） | 先全部依序填好 |
| 是否使用遞迴 | 是 | 否 |
| 呼叫堆疊 | 需要遞迴堆疊 | 不需要 |
| 適合使用情境 | 只計算少量指定的 `C(n,k)` | 需要大量或整表的 `C(n,k)` |

---

## 總結
- **查表版**：在遞迴過程中「邊算邊存」，較貼近數學定義。  
- **動態規劃版**：先建立完整表格，再直接查值，效能穩定且無遞迴風險。
