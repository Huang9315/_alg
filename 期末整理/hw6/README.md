
# 線性迴歸最佳化演算法比較


---

##  演算法說明與個人理解

### 1. 爬山算法 (Hill Climbing)
* **理解**：這是一種隨機搜尋法。像是蒙著眼睛在山坡上隨機踏出一小步，只要那一小步比現在的位置低（誤差更小），就往那裡走。
* **優點**：簡單易懂。
* **缺點**：容易陷入局部最優解，且效率較低。

### 2. 貪婪法 (Greedy Method)
* **理解**：在每一步中，觀察「上下左右」所有可能的固定步長位置，直接選取當下能讓誤差降最多的那個點。
* **與爬山的區別**：爬山是隨機試探，貪婪是窮舉局部可能性後選擇最優。

### 3. 陡峭下降法 (Steepest Descent / Gradient Descent)
* **理解**：這是最科學的方法。它利用微積分計算出函數下降最快的方向（梯度的反方向）。
* **個人見解**：它是「貪婪法」的數學進化版。與其漫無目的地試探步長，不如直接用數學公式算出斜率，順著斜率往下衝。

### 4. 改良法 (Improvement Method)
* **理解**：通常指在迭代過程中動態調整步長（例如一開始大步走，接近目標時小步走），或者是在隨機搜尋中加入動量（Momentum）來跳出局部最優。
